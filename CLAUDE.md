## プロダクト概要

故障したゲーミングPC（マウスコンピューター G-Tune HN-A：Ryzen 7 3700X／
RTX 2070 SUPER／ASRock X570 Phantom Gaming 4 ベースの OEM マザーボード）を
復活させるプロジェクト。step1 現状把握 → step2 部品調達（使える部品は流用）→
step3 組み立て・動作検証（動作しなければ step1 へ戻る）→ step4 復活完了、の順で
進める。全 step を通じて、作業の備忘録を HTML 文書として `docs/site/` に蓄積し、
トップページ（`index.html`）から参照できる形で公開可能にしておく。

### 厳守すべき制約

- 既存の使える部品はそのまま活用し、必要な部品のみを新たにそろえる（既存資源優先・ゼロコスト重視）
- 備忘録は `docs/site/` に HTML で残し、`index.html` から全文書へたどれるようにする
- 公開対象は `docs/site/` 配下のみ。既存 `docs/` 直下の 2 文書には製品番号等が
  含まれるため、公開物へ転記する際は個人情報・購入情報を除く
- `docs/G-Tune_HN-A_spec.html` は個体の製品番号とメーカー仕様の転載を含むため
  **Git 非追跡**（ローカル参照のみ。`.gitignore` で除外済み）
- 備忘録は文字だけにせず、写真・概念図・配線図などを併載する

### 完了条件

1. PC が安定して起動し、Windows が正常に動作する
2. 高負荷時（Steam ゲーム等）でも突然の画面消失・再起動が再発しないことを確認する
   （故障の発端が Steam ゲーム中のブラックアウト → 再起動のため）
3. 経過・原因・対処が `docs/site/` の HTML 備忘録に記録され、`index.html` から
   参照できる状態になっている

## 使用言語

プロジェクト配下での生成テキストはすべて日本語で書く。対象は以下を含む。

- ユーザーへのチャット返信、進捗報告、ターン末の要約
- `AskUserQuestion` の質問文・選択肢のラベルと説明
- コード内のコメント（`//`, `#`, `/* */`, docstring 等）
- コミットメッセージ、PR タイトル、PR 本文
- README、CLAUDE.md、設計メモ、その他のドキュメント類
- エラーメッセージやログ出力のうち、エンドユーザー向けに表示される文字列

例外として **英語のまま** とするのは次のみ（言語仕様・エコシステム慣習のため）。

- プログラミング言語のキーワード、標準ライブラリ／フレームワーク／パッケージの API 名
- 識別子（変数名・関数名・クラス名・ファイル名・ディレクトリ名など）
- 設定ファイルのキー名、環境変数名
