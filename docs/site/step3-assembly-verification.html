新トラブル — 旧 OS SSD（CFD PG3VNF 1TB）の認識不良

交換作業の前に、旧 OS 入り M.2 SSD の中身を確認しようと、SSD ケース （ロジテック LHR-LPNVW02UCD・USB 3.2 Gen2 対応の NVMe ケース）に入れて 別の Windows PC へ接続した。経過は次のとおり。

  1. 対処旧 OS SSD を SSD ケースに装着し、別の Windows PC へ USB 接続（2026-07-16）
  2. 正常いったんは認識し、エクスプローラーで中身も表示された
  3. 異常フォルダを開こうとすると数分待たされ、ようやく開く状態に
  4. 異常しばらくすると SSD 自体が認識されなくなった
  5. 異常「ディスクの管理」でも表示まで待たされ、「不明・初期化されていません」に。容量表示すら出ない（写真1）
  6. 対処「ディスクの初期化」ダイアログ（写真2）はキャンセル——初期化はしていない

初期化してはいけない理由：「初期化」は新しい空のパーティション管理情報を 書き込む操作で、実行するとデータ救出の難易度が一気に上がる。逆に言えば 「初期化されていません」という表示はデータが消えたという意味ではなく、 管理情報が読み取れていないだけの可能性がある。同じ理由で、フォーマットや chkdsk（チェックディスク）も救出が終わるまで実行しない。

考察：「読み込みが数分単位で停滞 → その後認識消失」という経過は、読み取りエラーの リトライを繰り返している SSD 側の不調（コントローラや NAND の異常）に典型的な パターン。ただし NVMe SSD は USB ケース内で高温になりやすく、ケースの発熱や USB ブリッジの相性でも似た症状が出ることがあるため、クロ確定は後述の NVMe 直結での再確認まで保留する。

切り分けへの影響：step1 で確認したとおり、起動ループは SSD を全部外しても再現した。 つまりこの SSD 不調は「PC が起動しない」原因ではなく、別の（あるいは同じ根本原因から 派生した）追加のトラブルであり、「マザーボード最有力」という見立ては変わらない。 一方で「根本原因が電源で、マザーボードや SSD が巻き添えになった」という筋も 考えられるため、第 2 段（電源交換）の構えは維持する。

方針の修正 — OS は P310 へ、旧 SSD は救出対象に

当初の搭載計画（step2 の図2）では旧 OS SSD を M.2_1 に載せてそのまま起動する予定だったが、信頼できないドライブから OS を起動するのは 検証の混乱のもとになる。追加購入なし（既存部品のみ）で成立する次の形に修正する。

OS ドライブ旧 CFD PG3VNF を流用 → Crucial P310 1TB に Windows 11 をクリーンインストール（M.2_1・Gen4 CPU 直結へ配置換え。M.2_1 にヒートシンクは付属しない——板の付属ヒートシンクは M.2_2 側）
旧 CFD PG3VNFM.2_1 に搭載 → 当面 PC に組み込まない。未バックアップの大事なデータがあるため「データ救出対象」として隔離保管し、通電は救出作業のときだけにする
M.2_2P310（ゲーム用）→ 空き。救出作業のときだけ CFD を一時装着する口として使う
P310 の中身インストール済みだったゲームは消えるが、Steam から再ダウンロードで復元できる
Windows ライセンスMicrosoft アカウント紐付けのデジタルライセンスなので、クリーンインストールでも再認証できる見込み（手順は step2 参照）

データ救出計画 — CFD PG3VNF から大事なデータを取り出す

旧 OS SSD には未バックアップの大事なデータが残っている。読めなくなりかけの ドライブは通電とリトライのたびに状態が悪化しうるため、 「読めている間に、優先度の高いものから、コピーだけ」が大原則になる。

  • 初期化・フォーマット・chkdsk を実行しない——管理情報を書き換えて救出を難しくする操作はすべて禁止
  • 書き込まない——救出は読み取り（コピー）だけ。SSD 内のファイルを開いて編集・保存しない
  • 通電は救出作業のときだけ——普段は取り外して保管する
  • 先に「救出優先リスト」を作る——書類・写真 → ブラウザのブックマーク → ゲームのセーブデータ（Steam クラウド対象外のもの）→ その他、の順に救う場所を紙に書き出しておく。コピー先の空き容量も確保しておく

第 1 手を「PC 復活後」に置くのは、マザー交換で確実に動く受け皿を先に作るため。 ただし第 1 段（マザー交換）で直らず第 2 段（電源交換）へ進むことになった場合は、 復活を待たずに第 2 手（別 PC ＋ケース）を先に試してよい。その際はケース付属の 放熱シートを貼り、熱がこもらない置き方で短時間勝負にする。復旧ソフトを使う場合は 「読み取り専用でスキャンするもの」を選び、復元先は必ず別のドライブにする。 専門業者は数万〜数十万円かかるため、データの価値と天秤にかけて判断する。

「エタノール配合」の手指消毒液を代用しない：実作業当日（2026-07-17）、 手元にあった手指の消毒液（写真3）をグリス拭きに使えないか検討したが、見送った。 この種の製品はグリセリンや中鎖脂肪酸トリグリセリド（＝油分）などの保湿成分を含み、 拭いたあとの金属面に油膜が残って熱伝導の妨げになる。ラベルに「エタノール」とあっても 溶剤として入っているだけで、中身は水ベースの消毒液——ほぼ何も残さず揮発する 無水エタノール（純度 99.5%）とは別物。ウェットティッシュ類も同じ理由で不可。 なければ乾拭きで十分（今回も乾拭きで対応）。

組み込み（TUF GAMING B550-PLUS）

  1. 新しい板を箱（帯電防止袋の上）に置き、CPU を装着する——CPU 角の▲マークをソケットの▲に合わせ、力をかけずに載せてレバーを倒す
  2. メモリ 2 枚をマニュアル推奨の A2・B2 スロットへ（カチッと両端のラッチが閉じるまで）
  3. P310 を M.2_1 に装着する——M.2_1 にヒートシンクは付属しない（付属ヒートシンクは M.2_2 側。そちらを使うときは裏の保護フィルムを剥がしてから）
  4. CPU に MX-4 を塗る——中央に米粒 1.5〜2 粒分（直径 4〜5mm のひと盛り）。ヘラで伸ばさなくてもクーラーの圧着で広がる。MX-4 は非導電性なので多少はみ出しても無害——少なすぎるより多すぎるほうがまし。クーラーは載せたら位置をずらさず、対角順に「全体を仮締め → もう 1 周で本締め」の 2 段階で均等に締める。最後にファンケーブルを CPU_FAN ヘッダへ接続（忘れると起動時に CPU Fan Error で止まる、挿し忘れの定番）
  5. I/O パネル（バックパネル）を先にケースへはめる——TUF GAMING B550-PLUS は板と別体。ケースに板を入れたあとでは取り付けられない、忘れやすさ No.1 の部品
  6. 板をケースに入れてネジ 9 本で固定し、撮っておいた写真を見ながら配線する——24 ピン／EPS 8 ピン／フロントパネル（電源スイッチ・LED はマニュアルのピン配置図参照）／USB ヘッダ／HD AUDIO／ケースファン
  7. GPU を一番上の x16 スロットに装着して補助電源を接続、Wi-Fi 子機を空きの x1 に装着してアンテナをつなぐ
  8. 図1 と写真を見ながら、挿し忘れ・仮止めのままのネジがないか最終確認する

配線の要点：撮っておいた写真は「どのケーブルか」を見分けるための資料で、 挿し先は新しい板のマニュアルのピン配置図が正（旧板とはヘッダの位置が違う）。 とくに EPS 8 ピン（CPU 電源）と GPU 補助電源（PCIE）はコネクタ形状が似ていて 取り違えやすい——ケーブル側の刻印「CPU」を確認し、切り欠きが合わないものは無理に 押し込まない。CPU 電源は 8 ピンのみで足りる（隣の 4 ピンは空きでよい）。フロントパネルは PWR SW・RESET SW が極性なし、PWR LED・HDD LED は極性ありだが、間違えても光らないだけで 壊れない。24 ピンはラッチが鳴るまで挿し込む——半挿しは「通電しない」の定番原因。

慎重に進めたい場合は、ケースに入れる前に「箱の上で最小構成テスト」を挟むと切り分けが 楽になる——CPU＋クーラー＋メモリ 1 枚＋GPU＋電源（24 ピン・EPS・GPU 補助）だけつなぎ、 マニュアルの電源スイッチピンをドライバーの先で短絡して起動、BIOS 画面が出るかだけ確認する。 3700X は内蔵 GPU がないため、このテストでも GPU とモニタ接続は必須。

  13. 起動14:40 UEFI（BIOS）画面が表示。CPU・メモリ 32GB・M.2_1 の P310 を認識（写真42・下の表）

メモリ表示について：POST の「DDR4-3200」はメモリ側の定格表示で、 D.O.C.P（XMP 相当）を有効化したという意味ではない可能性が高い。計画どおり、まずは定格（自動設定）の まま安定を確認し、D.O.C.P はデータ救出まで終えてから有効化する。実際の動作クロックは UEFI の Memory 項目で要確認。

気づき（次工程への申し送り）： ① BIOS が P310・32GB・CPU をすべて認識し、起動ループも暗転も POST では再発しなかった—— マザーボード交換（第 1 段）で改善した公算が大きい。ただし完了条件（高負荷で暗転が再発しないこと）は フェーズ 3 まで要確認で、電源交換の二段構えは残す。 ② この起動時はキーボード未接続だった（POST に「NO Keyboard Detected」）。 UEFI 設定と OS インストールには USB キーボードを接続してから進める。 ③ 旧板の HD_AUDIO は新板では AAFP 表記、前面カードリーダーは内部 USB （USB_E12・E34 の両方）で接続。旧板で使っていた USB ヘッダアダプタは新板では不要だった。 ④ SATA に内蔵 Blu-ray ドライブ（BU40N）がつながっていることも BIOS で確認できた。 ⑤ 板の固定はケース側スタンドオフのある 7 か所のみ（ATX の 9 か所のうち 2 か所はスタンドオフがなく未固定・写真29）。 実用上は問題ないと判断したが、24 ピンを挿すときは基板がややぐらつくので注意。気になる場合はスタンドオフを 2 か所 足せば全 9 か所を固定できる。

次はフェーズ 2＝P310 への Windows 11 クリーンインストール——その顛末は 次の「作業記録③」へ。ライセンス認証は当初「Microsoft アカウントで再認証」を見込んでいたが、 実際には OEM ライセンスの制約でつまずいた（作業記録③で詳述）。続けてフェーズ 3（ドライバ導入と 高負荷検証）、フェーズ 4（旧 SSD からのデータ救出）へと進む。

作業記録③ — OS 導入とライセンス認証（2026-07-22 実施）

新板（TUF GAMING B550-PLUS）に組み替えた PC へ、いよいよ Windows 11 を導入する。 インストールメディアは別 PC で作った BUFFALO のポータブル SSD。周辺機器を すべてつないで電源を入れると、9:37 の POST で全デバイスを認識し、BIOS を通過して Windows 11 セットアップへ——10:36 にはデスクトップへ到達し、フェーズ 2（OS 導入）を 完了した。ただし途中で2 つのつまずきがあった。ひとつは インストール先ディスクの選択エラー（流用した P310 に旧パーティションが残存）、 もうひとつはライセンス認証——ここが今回いちばんの学びで、 OEM ライセンスはマザーボード交換では移行できず、最終的にリテール版 Windows 11 Home を 購入して認証することになった（下の図4）。

  1. 開始9:19 周辺機器（有線マウス・キーボード・スピーカー・LAN・HDMI）とインストール USB（BUFFALO）を接続して電源投入（写真43・44）
  2. 認識9:37 POST で全デバイスを認識——キーボードも認識（作業記録②の「NO Keyboard Detected」解消）。画面の「Press F1 to Run SETUP」に従い F1 で UEFI へ（写真45・46）
  3. 対処9:42〜9:45 UEFI（EZ Mode）：ずれていた BIOS 時計を修正、表示を日本語化、D.O.C.P は Disabled のまま、起動優先順位＝UEFI OS (BUFFALO) を確認して保存＆終了（写真47〜49）
  4. 完了9:49〜9:52 Windows 11 セットアップ：言語（日本語）→「Windows 11 のインストール（すべて削除＝クリーンインストール）」→ プロダクトキー「ありません」→ Windows 11 Home → ライセンス条項に同意（写真50〜53）
  5. 異常9:53 ★つまずき①：インストール場所の選択でエラー——流用した P310（ディスク 0）に旧パーティション（MSR＋SSD 931.5GB／EFI システムパーティションなし）が残存していた（写真54・55）
  6. 対処9:58〜9:59 ディスク 0 のパーティションを全削除して未割り当て（931.5GB）に → 解決（Windows が EFI・MSR・主要区画を自動作成）（写真56・57）
  7. 完了9:59〜10:03 インストール準備完了 → インストール（10%→51%→再起動→「インストール中 33%」）（写真58・59）
  8. 対処〜10:04 ファイルコピー完了後、インストール USB（BUFFALO）を抜く（写真60）
  9. 完了10:06〜10:32 OOBE（Out Of Box Experience＝箱から出して最初に行う Windows の初期設定）：地域＝日本、キーボード＝Microsoft IME、Microsoft アカウントでサインイン、PIN 作成、プライバシー設定（必須のみ）（写真61〜63）
  10. 到達10:33〜10:36 PC バックアップ復元は「新しい PC として」、宣伝類はスキップしてデスクトップ到達。ASUS の Armoury Crate（RGB 管理サービス）は不要のため見送り（写真64・65）
  11. 異常10:58〜11:09 ★つまずき②：ライセンス認証できない——「リンクされたデバイスが見つかりません」。devices に旧機 MOUSE6（HN-AX570） は載るが OEM は移行不可。MS サポート無料相談も不可（写真66・67）
  12. 解決リテール版 Windows 11 Home を購入して認証成功——以後はマザー交換でも再認証できる（下の図4）

気づき（教訓）： ① UEFI へは F1——新板の初回は BIOS 設定がリセット状態のため「Press F1 to Run SETUP」が出る（DEL/F2 でも入れる）。 ② 流用ドライブは残存パーティションに注意——インストール先でエラーが出たら、そのディスクを 全削除して未割り当てにすれば、Windows が起動に必要な区画を作り直す（今回の P310 が該当）。 ③ ライセンスの落とし穴——OEM プリインストール機のライセンスはマザーボードに紐づき、 マザー交換では移行できない（下の図4）。当初計画の「Microsoft アカウントで再認証」は誤りだった。 ④ インストールメディアは BUFFALO のポータブル SSD で問題なく認識できた。

その後（ドライバとアカウント）：デスクトップ到達後、GPU は NVIDIA の 「グラフィックスドライバーのみ」（NVIDIA App は入れない）、AMD の チップセットドライバを導入した。デバイスマネージャーに黄色い「！」は出ていない。 仕上げに、ユーザーフォルダ名を旧 PC と同じにするため——Microsoft アカウントで作成すると フォルダ名がメールアドレスの先頭になり、既存フォルダのリネームもできない—— 希望名でローカルアカウントを新規作成して切り替えた（旧 Microsoft アカウントは正規手順で削除）。

これでフェーズ 2（OS 導入）は完了。次はフェーズ 3c＝安定性検証——HWiNFO で温度を 監視しつつ、Steam を入れ直して発端のゲームを 1〜2 時間プレイし、暗転・再起動が 再発しないかを確かめる。ここを通れば本プロジェクトの完了条件を満たす。続けてフェーズ 4（旧 SSD の データ救出）へ。

動作検証チェックリスト

電源投入からゲーム負荷まで、4 つのフェーズで段階的に確認する。フェーズ 3 まで 通れば本プロジェクトの完了条件（安定起動＋高負荷で暗転再発なし）を満たす。

フェーズ 1
起動確認		 2026-07-20 通過（作業記録②）。電源 ON → 板上の Q-LED（CPU → DRAM → VGA → BOOT）が順に消灯して UEFI（BIOS）画面が表示されるか。UEFI 上で CPU（Ryzen 7 3700X）とメモリ 32GB の認識、CPU 温度（アイドルでおおむね 40〜60℃）を確認。→ BIOS が CPU・32GB・P310 を認識して起動。残りは UEFI 上での温度確認
フェーズ 2
OS 導入		 2026-07-22 通過（作業記録③）。インストール USB（BUFFALO）から起動 → Windows 11 Home をクリーンインストール（インストール先は P310 の旧パーティションを全削除して解決）→ OOBE。ライセンス認証は当初「Microsoft アカウントで再認証」を見込んでいたが、OEM ライセンスはマザー交換で移行できず不可 → リテール版 Windows 11 Home を購入して認証成功（顛末は上の作業記録③・図4）。あわせて GPU／チップセットドライバを導入し、ローカルアカウントへ切り替えた
フェーズ 3
安定性検証		 Windows Update 完走・チップセット／GPU／LAN ドライバ導入 → アイドル 30 分と再起動数回で異常がないこと → Steam を入れ直し、発端となったゲームを 1〜2 時間プレイして暗転・再起動が再発しないことを確認（温度もモニタリングする）
フェーズ 4
データ救出		 安定を確認できたら、M.2_2 に CFD を一時装着してデータ救出（図2 のフロー）。救出後にメモリ設定 D.O.C.P（DDR4-3200）を有効化して仕上げる

メモリは最初は定格（自動設定）のまま起動し、安定を確認してから D.O.C.P を有効にする。 変数を 1 つずつ増やすことで、万一不調が出たときに原因を切り分けやすくする。

動かなかった場合 — 第 2 段（電源交換）へ

マザーボードを交換してもCPU・DRAM LED 点灯 → 数秒で再起動のループが続く場合は、 step1 の見立てどおり次点の容疑・電源ユニットの交換（650W 以上）に進む。 これは進行フローの「step3 → step1 へ戻る」ループにあたる。

  • 症状が交換前と同じ（CPU・DRAM 点灯 → 再起動ループ）→ 電源が本命に昇格。第 2 段へ
  • 症状が交換前と変わった（点灯する LED や挙動が違う）→ 組み立てミスの可能性から先に潰す。配線写真と図1 で全接続を再点検
  • まったく通電しない → 24 ピン・EPS の挿し込み、電源背面スイッチ、フロントパネルの電源スイッチピンの順に確認

新品マザーボードの初期不良という可能性もゼロではない。購入店の初期不良対応期間内に フェーズ 1（BIOS 起動確認）まで済ませておくと、万一のときに交換を受けられる。
